# Chronexa Deployment Guide

This guide walks through deploying Chronexa to **Vercel (frontend)** and **Railway (backend & database)** with automatic CI/CD via GitHub Actions.

## 📊 Architecture Overview

```
GitHub Repository
    ↓
GitHub Actions CI/CD Pipeline
    ├→ Vercel (Frontend) - React + TypeScript
    └→ Railway (Backend) - Express.js + PostgreSQL + Redis
```

## 🚀 Deployment Platforms

| Component | Platform | Cost | Benefits |
|-----------|----------|------|----------|
| Frontend | **Vercel** | Free tier | Zero-config deployment, optimal for Next.js/React/Vite |
| Backend | **Railway** | Free $5/month | All services (PostgreSQL, Redis, Express.js) |
| Database | Railway PostgreSQL | Free tier | Included in Railway, 100MB+ storage |
| Cache | Railway Redis | Free tier | Included in Railway |
| Storage | Railway MinIO | Free tier | Included in Railway |

## 📋 Prerequisites

Before deploying, ensure you have:

- ✅ GitHub account with your Chronexa repo already pushed
- ✅ Vercel account (free tier)
- ✅ Railway account (free tier)
- ✅ All environment variables documented in `.env.example`

## ⚙️ Step 1: Set Up Vercel (Frontend)

### 1.1 Create Vercel Project

1. Go to [vercel.com](https://vercel.com) and sign in with GitHub
2. Click **Add New** → **Project**
3. Find your **Chronexa** repository and click **Import**
4. Configure settings:
   - **Framework Preset**: Vite
   - **Root Directory**: `./client`
   - **Build Command**: `npm run build`
   - **Output Directory**: `client/dist`
   - **Install Command**: `npm install`

### 1.2 Add Environment Variables to Vercel

1. In Vercel project settings → **Environment Variables**
2. Add these variables:

```
VITE_BACKEND_URL=https://your-railway-backend.up.railway.app/api
```

Replace `your-railway-backend` with your actual Railway backend service URL (you'll get this in Step 3).

### 1.3 Configure Custom Domain (Optional)

1. In Vercel settings → **Domains**
2. Add your custom domain or use the default `chronexa.vercel.app`

### 1.4 Enable Automatic Deployments

- Deployments are **automatic** on every push to `main` branch
- For manual deploy: Click **Deploy** in Vercel dashboard

## ⚙️ Step 2: Set Up GitHub Actions Secrets

### 2.1 Get Vercel Tokens

1. Go to [vercel.com/account/tokens](https://vercel.com/account/tokens)
2. Create a new token and copy it
3. In your GitHub repo → **Settings** → **Secrets and variables** → **Actions**
4. Add these secrets:

```
VERCEL_TOKEN=<your_vercel_token>
VERCEL_ORG_ID=<your_vercel_org_id>
VERCEL_PROJECT_ID=<your_vercel_project_id>
```

**Finding Vercel IDs:**
- In Vercel project → Settings → General
- Copy "Project ID" and "Org ID"

### 2.2 Get Railway Token

1. Go to [railway.app](https://railway.app) and sign in with GitHub
2. Click **Account** → **Tokens**
3. Create new token and copy it
4. In GitHub repo → **Settings** → **Secrets** → Add:

```
RAILWAY_TOKEN=<your_railway_token>
```

## ⚙️ Step 3: Set Up Railway (Backend)

### 3.1 Create Backend Service

1. Go to [railway.app](https://railway.app)
2. Click **+ New Project**
3. Choose **Deploy from GitHub**
4. Select your **Chronexa** repository
5. Set these variables:
   - **Service Name**: `chronexa-backend`
   - **Root Directory**: `.` (root)
   - **Build Command**: `npm install`
   - **Start Command**: `npm run server`

### 3.2 Create PostgreSQL Database

In the same Railway project:

1. Click **+ Add Service** → **Database** → **PostgreSQL**
2. Railway auto-generates connection string as `DATABASE_URL`
3. The database should be auto-linked to your backend service

### 3.3 Create Redis Cache

In the same Railway project:

1. Click **+ Add Service** → **Cache** → **Redis**
2. Railway auto-generates connection string as `REDIS_URL`
3. Auto-linked to your backend service

### 3.4 Add Backend Environment Variables

In Railway project → Variable list, add these:

```env
# Core
NODE_ENV=production
BACKEND_PORT=3000

# Database (usually auto-generated by Railway)
DATABASE_URL=postgresql://...
DB_SYNCHRONIZE=false

# JWT Tokens (Generate random strings - at least 32 chars each)
JWT_REFRESH_SECRET=your_long_random_refresh_secret_here_min_32_chars
JWT_ACCESS_SECRET=your_long_random_access_secret_here_min_32_chars

# Email (Use your Resend API key from Step 3.5)
RESEND_API_KEY=re_your_resend_api_key_here
MAIL_FROM=Chronexa <noreply@chronexa.com>

# Redis (usually auto-generated by Railway)
REDIS_URL=redis://...

# Frontend URL (Get from Vercel deployment)
FRONTEND_URL=https://your-chronexa.vercel.app

# MinIO (Railway auto-deploys this)
OBJECT_STORAGE_ENDPOINT=localhost
OBJECT_STORAGE_ACCESS_KEY=minioadmin
OBJECT_STORAGE_SECRET_KEY=minio12345
OBJECT_STORAGE_PORT=9000
OBJECT_STORAGE_URL=https://your-minio.up.railway.app
```

**⚠️ Important:** Do NOT commit these secrets to git. Keep them in Railway's secure vault only.

### 3.5 Get Gmail App Password

For email functionality in production:

1. Enable 2FA on your Gmail account
2. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
3. Create app password for "Mail" on "Other (custom name)"
4. Copy the 16-character password and add as `SMTP_PASS` in Railway

### 3.6 Deploy & Get Backend URL

1. Click **Deploy** in Railway
2. Once deployed, click on your backend service
3. Copy the **Public URL** (e.g., `https://chronexa-backend.up.railway.app`)
4. Update Vercel `VITE_BACKEND_URL` with: `https://your-backend.up.railway.app/api`

## 🔄 Step 4: CI/CD Pipeline Configuration

### 4.1 How Auto-Deployment Works

Every time you push to `main` branch:

1. **GitHub Actions runs**:
   - Lints your code
   - Builds frontend
   - Builds backend TypeScript
   - Tests connections

2. **If successful**:
   - Vercel deploys frontend to `chronexa.vercel.app`
   - Railway deploys backend
   - Your app updates live in ~2-3 minutes

### 4.2 View Deployment Status

**Vercel:**
- Dashboard → Deployments tab
- Shows all deployments with timestamps and status

**Railway:**
- Project → View deployments
- Shows build & deployment logs

**GitHub Actions:**
- Repository → Actions tab
- See CI/CD pipeline status for each push

### 4.3 Rollback a Deployment

**Vercel:**
- Go to Deployments → Click previous version → **Promote to Production**

**Railway:**
- Go to Deployments → Select previous build → **Deploy**

## 🐛 Troubleshooting

### Deployment Fails on GitHub Actions

**Error: "VERCEL_TOKEN not found"**
- Go to GitHub → Settings → Secrets → Check if `VERCEL_TOKEN` is set
- Verify token is not expired (recreate if needed)

**Error: "Build failed"**
- Click build link to see full logs
- Common causes:
  - Missing environment variables
  - TypeScript errors in backend
  - Missing dependencies in package.json

### Vercel Shows Blank Page

**Issue: Frontend loads but is blank**
1. Open browser DevTools (F12)
2. Check Console tab for errors
3. Check if `VITE_BACKEND_URL` points to correct Railway backend
4. Verify backend is running (test with curl)

**Fix:**
```
- Redeploy Vercel
- Update VITE_BACKEND_URL
- Clear browser cache
```

### Backend Connection Errors

**Error: "Cannot connect to database"**
1. Check PostgreSQL is running: Railway dashboard → PostgreSQL service
2. Verify `DATABASE_URL` in Railway variables
3. Check if `DB_SYNCHRONIZE=false` for production

**Error: "Redis not available"**
1. Check Redis service in Railway dashboard
2. Verify `REDIS_URL` is set in environment variables
3. If new deployment, Redis may need 30 seconds to initialize

### Emails Not Sending

**Error: "Resend API error" or email not sending**
1. Verify Resend API key is correct: format should start with `re_`
2. Check RESEND_API_KEY is set in Railway env vars
3. Ensure sender email is verified in Resend dashboard (for production domains)
4. Test locally first before deploying

**Fix:**
```env
RESEND_API_KEY=re_your_actual_api_key
MAIL_FROM=noreply@chronexa.com
```

**Get Resend API Key:**
1. Sign up at https://resend.com
2. Go to API Tokens section
3. Create new token and copy it
4. Add to Railway environment variables as `RESEND_API_KEY`

### Custom Domain Issues

**SSL Certificate Error**
- Vercel auto-generates SSL certificates (usually 5 min)
- Wait 5 minutes and force refresh (Ctrl+Shift+R)

**DNS Not Pointing Correctly**
- Check Vercel → Domains → see exact CNAME records needed
- Update DNS provider with exact records

## 🔒 Production Security Checklist

Before going live:

- [ ] All JWT secrets are **random and strong** (32+ chars)
- [ ] Database has **admin password set** (Railway auto-generates)
- [ ] Email credentials **never committed** to git
- [ ] FRONTEND_URL in Railway matches **actual domain**
- [ ] CORS headers allow **only your frontend domain**
- [ ] `DB_SYNCHRONIZE=false` in production
- [ ] Vercel preview deployments require **approval** (optional)
- [ ] Railway backend service has **restart policy** set
- [ ] GitHub Actions secrets are **all configured**
- [ ] Error logs don't expose sensitive info (**review logs**)
